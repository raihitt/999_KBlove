## 概要

- IRBANKから任意の日本株銘柄の財務指標を取得し、今年度を除いた直近10期分を統合して正規化CSVに出力するPython 3スクリプトを開発する。
- pandas・requests・BeautifulSoupの3ライブラリを必須とし、スクレイピング、整形、派生指標計算、データ保存を関数分割するモジュール構成を採用する。
- セレクタの切り出し、軽量なバリデーション、実用的なログ出力を通じてIRBANK側のレイアウト変更に耐える保守性を確保する。

## ステークホルダーと利用者

- 定量分析用データセットを整備したい個人投資家・アナリスト。
- 指標拡張や他市場対応を担うエンジニア。スクレイピングロジックに対する簡潔な補足コメントを期待。

## 入力

- 必須パラメータ: `--stock-code`（4桁の銘柄コード）。
- 任意パラメータ: 出力パス（既定は `data/irbank_{code}.csv`）、リクエストタイムアウト、リトライ回数、User-Agent文字列、ログレベルなど。

## データソース

```
results:         年度, EPS, BPS, 自己資本比率
profitability:   売上高総利益率, 売上高営業利益率, 売上高純利益率
efficiency:      ROE, ROA, 売上高販売管理費比率, 総資産回転率
safety:          手元流動性比率, 流動比率, 当座比率, 営業CF対流動負債比率, 有利子負債月商倍率
value:           時価総額, 株価
per:             PER
pbr:             PBR
pl?num=20:       売上高, 売上原価, 売上債権
bs?num=20:       仕入債務, たな卸資産, 固定資産, 固定負債, 自己資本
cf?num=20:       営業CF
debt?num=20:     有利子負債（無い場合は bs から補う）
```

- 補助項目が取得できない場合は欠損値（`None` → CSVでは空欄、レポートでは `-` 表記）として扱い、警告ログを残す。代替ソースは使用しない。

## 年度の扱い

- IRBANKに掲載される予想値（「予」「(E)」「FY****予想」など）や四半期値は除外する。
- 確定済み年度のみを対象とし、最新年度が当期の場合はスキップ。残りから過去に遡って最大10期分を取得する。
- 年度キーは西暦（例: `2023`）に統一し、重複行は除去する。

## 単位と値の正規化

- すべての金額は円に換算する。
  - `兆円` = 1e12 円
  - `億円` = 1e8 円
  - `百万円` = 1e6 円
  - `千円` = 1e3 円
  - `円` = 1 円
- ％表示は `%` を取り除き数値化（例: `12.5%` → `12.5`）。
- 桁区切りや括弧付きマイナス（例: `(1,234)`）は符号を維持したまま数値化する。

## 機能要件

- robots.txtを遵守し、カスタムヘッダと指数的バックオフ付きのリトライで各ページを取得する。アクセス間隔は1〜2秒程度のジッターを入れて調整する。
- 年度テーブルを解析し、列名を日本語カノニカル表記に正規化。予想行・四半期行をフィルタリングした上で、年度キーを整形する。
- 年度をキーに各データセットをマージし、年度昇順に並べ替えた後で最新10期（今年度除外）に切り詰める。
- 派生指標は必要項目が揃った場合のみ算出し、不足時は `None` を設定してデバッグログに理由を残す。
- 定義済みカラム順のDataFrameをUTF-8 CSV（インデックスなし）として保存し、必要に応じて呼び出し元へDataFrameを返す。

## 派生指標

```
売上債権回転期間 = (売上債権 / 売上高) * 365
仕入債務回転期間 = (仕入債務 / 売上原価) * 365
たな卸資産回転期間 = (たな卸資産 / 売上原価) * 365
キャッシュ・コンバージョン・サイクル = 売上債権回転期間 + たな卸資産回転期間 - 仕入債務回転期間
固定比率 = (固定資産 / 自己資本) * 100
固定長期適合率 = (固定資産 / (自己資本 + 固定負債)) * 100
負債資本倍率(D/Eレシオ) = 有利子負債 / 自己資本
営業CF対有利子負債倍率 = 有利子負債 / 営業CF
PSR(株価売上高倍率) = 時価総額 / 売上高
```

- 0除算が発生する場合は `None` を返し、警告ログを出力する。

```
年度, 売上高総利益率, 売上高営業利益率, 売上高純利益率, ROE, ROA,
売上高販売管理費比率, 総資産回転率, 売上債権回転期間, 仕入債務回転期間,

固定長期適合率, 自己資本比率, 負債資本倍率(D/Eレシオ), 営業CF対有利子負債倍率,
時価総額, 株価, PER, PBR, PSR(株価売上高倍率), EPS, BPS
```

- 数値は小数で保持し、割合は `%` を外した値（例: 12.5）として保存する。

## 処理フロー

- `main()` で引数を解釈 → `fetch_all_metrics(stock_code)` で各ページ取得 → `merge_metrics(datasets)` で結合 → `compute_derived_fields(df)` で派生指標算出 → `persist(df, path)` で保存。
- 実行中はHTMLをメモリキャッシュし、同一ページへの重複リクエストを避ける。ディスクキャッシュは設定で明示的に有効化しない限り利用しない。
- ログ出力はINFOで進捗、DEBUGで詳細パース内容を報告できるようにする。

## エラーハンドリングと例外ケース

- 200以外のレスポンスやネットワークエラー時は設定回数までリトライし、それでも失敗した場合は分かりやすいメッセージで終了する。
- 想定テーブルが存在しない、または列名がマッチしない場合は警告を出して該当指標をスキップし、処理自体は継続する。
- 十分な年度データが存在しない場合は取得できた年度のみを出力し、期待数との差分をログに残す。
- 書き出し前に必須カラムの存在を検証し、欠落があれば例外を送出する。

## 依存関係と実行環境

- Python 3.9以上。
- pandas >=1.5, requests >=2.31, beautifulsoup4 >=4.12。
- 要求仕様に合わせて `requirements.txt` を `~=` でバージョン指定する。
- 認証・プロキシ環境は想定しない。
